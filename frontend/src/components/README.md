# /src/components/kit/ ディレクトリについて

## 概要 (Overview)

このディレクトリは、本アプリケーションにおける**新しいUIコンポーネントの唯一の置き場所（Single Source of Truth）**です。

今後のすべての新機能開発は、原則としてこの `kit` ディレクトリ内のコンポーネントを利用して行います。

## 背景 (Background)

このプロジェクトには、歴史的な経緯から複数のUIコンポーネントディレクトリが存在します。混乱を避け、今後の開発をクリーンに保つため、この `kit` ディレクトリが新設されました。

- **ver1: (`ui/`, `base/`)**

  - プロジェクト初期のUIコンポーネント群です。
  - 現在は完全に凍結されており、バグ修正以外の変更や新規利用は**禁止**されています。

- **ver2: `elements/`, `templates/`**

  - `ver1`からの移行期に作成されたコンポーネント群です。
  - このディレクトリも現在ではレガシー（移行対象）と見なされ、新規コンポーネントの追加は行いません。

- **ver3: `kit/` (このディレクトリ)**
  - 新しいUIライブラリ（Hero UI）の導入に伴い、これまでの技術的負債や命名の混乱から完全に切り離された「聖域」として作成されました。
  - `ui`や`elements`といった既存の名前を再利用すると混乱を招くため、新しい名前空間として`kit`が採用されました。

## ルールとガイドライン (Rules and Guidelines)

1.  **新しいUIコンポーネントは必ずここに配置する**

    - アプリケーションで共通利用するUI部品は、すべてこのディレクトリに追加してください。

2.  **新規開発では `kit` のコンポーネントのみを利用する**

    - 新しいページや機能の開発では、必ずこのディレクトリ内のコンポーネントを使用してください。

3.  **レガシーコンポーネントの新規利用は禁止**

    - `/components/ui/・/components/base/` および `/components/elements/・/components/templates/` 内のコンポーネントを新規に利用することは**固く禁止**します。

4.  **コンポーネントの責務を小さく保つ**
    - `kit` に追加するコンポーネントは、特定のビジネスロジックに依存しない、汎用的で再利用可能な状態にしてください。

## 既存コンポーネントからの移行について (Migration)

このプロジェクトの長期的な目標は、`/components/ui/・/components/base/` および `/components/elements/・/components/templates/` 内の古いコンポーネントを完全に削除することです。

ただし、「リファクタリングのためだけの時間」を設けるのではなく、日々の開発業務の中で段階的に移行を進めていきます。

**移行の進め方:**

- 既存のページや機能に手を入れる（機能追加・改修・バグ修正など）際に、その影響範囲で使われている古いコンポーネントを、`kit` の新しいコンポーネントに置き換えてください。
- 置き換えによってどこからも参照されなくなった古いコンポーネントは、その都度削除します。
